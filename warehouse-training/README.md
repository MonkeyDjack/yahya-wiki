# Пособие «Заявки на склад ХОЗ» — обучающий сайт YAHYA

Внутренний обучающий справочник для сотрудников сети YAHYA (продавцы, бариста). Учит правильно создавать заявки на склад ХОЗ (склад Ибраимова) в 1С и показывает, **что есть на складе** и **как это заказывать**. Mobile-first: открывается с телефона прямо на смене; на компьютере — раскладка с боковым меню.

Оформление — фирменное YAHYA («молочный шоколад»): тёплый коричневый фон, кремовый текст, золотые акценты, шрифт Montserrat.

---

## Что внутри

**4 основных экрана + карточка товара:**

1. **Главная** — меню: поиск, два больших входа («Как создать заявку», «Что есть на складе») и плитки по складам 1С.
2. **Как создать заявку** — пошаговый гайд из 6 шагов с прогресс-баром и кнопками Назад/Дальше. Объясняет: где в 1С создать заявку, как выбрать склад-отправитель и филиал-получатель (Бар/Магазин), как искать номенклатуру и как писать количество (штучно или упаковкой — пример «рукав = 27 шт → "1 рукав"»).
3. **Что есть на складе** — каталог из **186 позиций** с реальными фото. Фильтры по складам 1С: Склад МБП · Склад Прочее · Склад Сырьё · Склад Упаковка · Оборудование. Внутри — разделы (Чашки, Стаканы, Тарелки и т.д.) и счётчик найденных позиций.
4. **Карточка товара** — главный обучающий элемент. Крупное фото, **точное название как в базе 1С** (чтобы искали без ошибок), код номенклатуры, бейдж приёмника (Бар/Магазин), блок «Как заказывать» (единица измерения, фасовка, что писать в заявке) и кнопка **«Скопировать название»**.
5. **Поиск** — живой поиск по названию и коду, с дружелюбной заглушкой при пустом результате.

Бейджи приёмника: **Бар** — золотой, **Магазин** — зелёный.

## Как добавить как страницу на сайт

Папка самодостаточна и переносима — все пути относительные (`./…`).

**Вариант 1 — отдельная страница.** Загрузите папку `warehouse-training/` на сайт целиком (например в `/sklad/`) и открывайте `/sklad/index.html`. Поставьте ссылку из меню сайта.

**Вариант 2 — встроить через iframe:**
```html
<iframe src="/sklad/index.html" style="width:100%;height:100vh;border:0" title="Склад ХОЗ"></iframe>
```

**Что нужно для работы:** интернет (подгружаются React, Babel и иконки Lucide с CDN). Фото, шрифты и стили лежат внутри папки.

> Babel компилирует JSX в браузере — для внутреннего инструмента этого достаточно. При желании можно собрать через сборщик для продакшена.

## Как обновлять данные

Все позиции — в **`catalog.json`**. Каждая запись:

```json
{
  "n": 7,
  "part": "upakovka",                 // склад: posuda | hoz | syrye | upakovka | oborud
  "section": "Стаканы",               // раздел внутри склада
  "exact": "Стакан YAHYA бумаж…",      // точное название из 1С (показывается крупно)
  "code": "537",                       // код номенклатуры
  "receiver": "Бар",                   // приёмник: Бар / Магазин
  "unit": "шт",                        // единица измерения
  "flag": "",                          // "" | "check" (уточнить) | "add" (нет в базе)
  "img": "upakovka-007.jpg"            // файл фото в assets/warehouse/
}
```

- **Добавить товар:** добавьте объект в `catalog.json` и положите фото в `assets/warehouse/` с тем же именем, что в поле `img`.
- **Фасовка** (рукав/коробка/пачка) задаётся в `data.js` → объект `PACK`. Сейчас заполнены примеры для стаканов, крышек и пакетов; остальное показывается как «штучно». Когда фасовки утвердят — допишите их туда или добавьте поле в `catalog.json`.
- **Приёмник Бар/Магазин** в исходном Word помечен как предположительный — проверьте перед публикацией.
