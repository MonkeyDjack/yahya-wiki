# Каталог склада ХОЗ — что нужно доправить

## ⚠️ Проблема: разные товары под одним кодом (Посуда + Оборудование)

В исходном каталоге (`catalog.json`) при сборке из DOCX **разным предметам проставили одно
название и один код 1С**, а в нашем каталоге к ним привязаны **разные фото**. Проверено
глазами по фото: напр. под кодом `7143 «Соусник в ассортименте»` лежат 6 разных предметов
(соусник, миска, рифлёная кокотница, стеклянная креманка, квадратные блюдца, чашка).

**Затронуты только склады Посуда (10 групп) и Оборудование (2 группы). Сырьё / Упаковка /
Хоз — чистые** (каждая позиция с уникальным именем и кодом).

### Что сделать
Каждому фото вернуть **своё** название и код из 1С (не угадывать — сверять с 1С / Эмилем),
затем убрать общий фальшивый код. После — проставить корректный `reg` (nom/tov) по коду.

### Источник для сверки
`D:\Посуда с фото (1) (1) (1) (2) (1).xlsx` — инвентаризации по филиалам
(листы: АМ=Азия Молл, Эркиндик, Ибраимова, ЦУМ, Панфилова) с фото, названиями и
количествами. По фото можно опознать предметы и сверить названия.
